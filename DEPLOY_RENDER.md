## 🗄️ Bước 2: Setup MySQL Database

### Option 1: Dùng Blueprint (Tự động)

Blueprint sẽ tự động tạo MySQL database. Skip xuống Bước 3.

### Option 2: Tạo thủ công (Nếu không dùng Blueprint)

1. **Vào Dashboard** → **"New +"** → **"PostgreSQL"** (hoặc MySQL nếu có)
2. **Cấu hình**:
   ```
   Name: lms-db
   Database: lms_db
   User: lms_user
   Region: Singapore (gần Việt Nam nhất)
   Plan: Free
   ```
3. Click **"Create Database"**
4. Chờ 2-3 phút khởi tạo
5. **Lưu connection info**:
   - Host: `dpg-xxxxx-a.singapore-postgres.render.com`
   - Port: `5432`
   - Database: `lms_db`
   - Username: `lms_user`
   - Password: `<auto-generated>`

### 2.1. Import dữ liệu ban đầu

```bash
# Connect vào database (dùng psql hoặc MySQL client)
psql -h dpg-xxxxx.singapore-postgres.render.com -U lms_user -d lms_db

# Import SQL file
\i lms_db_backup.sql
```

## 🐳 Bước 3: Deploy Backend

### 3.1. Via Blueprint (Recommended)

1. Render detect `render.yaml` và hiển thị preview
2. Review các services:
   - ✅ `lms-backend` (Web Service)
   - ✅ `lms-db` (MySQL Database)
   - ✅ `lms-redis` (Redis)
3. Click **"Apply"**
4. Chờ 5-10 phút để:
   - Tạo database
   - Build Docker image
   - Deploy backend

### 3.2. Via Web Service (Manual)

Nếu không dùng Blueprint:

1. **New Web Service**:

   ```
   Name: lms-backend
   Environment: Docker
   Region: Singapore
   Branch: main
   Dockerfile Path: lms-backend/Dockerfile
   Docker Context: lms-backend
   ```

2. **Instance Type**:

   ```
   Plan: Free (512MB RAM, shared CPU)
   ```

3. **Environment Variables** (xem Bước 4)

4. Click **"Create Web Service"**

## 🔐 Bước 4: Environment Variables

### 4.1. Required Variables

Vào **Environment** tab và thêm:

```bash
# Spring Profile
SPRING_PROFILES_ACTIVE=prod

# Database (lấy từ lms-db connection info)
DB_URL=jdbc:mysql://dpg-xxxxx.singapore-postgres.render.com:5432/lms_db
DB_USERNAME=lms_user
DB_PASSWORD=<from-database-credentials>

# JWT Secret (generate 32+ bytes random string)
APP_JWT_SECRET=your-random-32-bytes-secret-key-change-this

# Redis (lấy từ lms-redis connection info)
REDIS_HOST=red-xxxxx.singapore-redis.render.com
REDIS_PORT=6379

# JVM Options (optimize cho 512MB RAM)
JAVA_OPTS=-Xms256m -Xmx512m -XX:MaxRAMPercentage=75.0 -XX:+UseG1GC

# Optional: Gemini AI (nếu dùng chatbot)
GEMINI_API_KEY=your-api-key-here

# Optional: Email (nếu dùng password reset)
MAIL_HOST=smtp.gmail.com
MAIL_PORT=587
MAIL_USERNAME=your-email@gmail.com
MAIL_PASSWORD=your-app-password
```

### 4.2. Generate JWT Secret

```bash
# PowerShell
-join ((48..57) + (65..90) + (97..122) | Get-Random -Count 32 | % {[char]$_})

# Hoặc dùng online: https://randomkeygen.com/
```

### 4.3. Sync Environment Variables

Sau khi thêm/sửa env vars:

1. Click **"Save Changes"**
2. Service sẽ tự động **redeploy**

## 🔗 Bước 6: Connect Frontend với Backend

### 6.1. Update Frontend Environment

Sửa `lms-frontend/src/environments/environment.prod.ts`:

```typescript
export const environment = {
  production: true,
  apiBaseUrl: "https://your-backend-name.onrender.com/api",
};
```

### 6.2. Update CORS

Sửa `lms-backend/src/main/java/com/ibizabroker/lms/configuration/WebSecurityConfiguration.java`:

```java
@Bean
CorsConfigurationSource corsConfigurationSource() {
    CorsConfiguration config = new CorsConfiguration();
    config.setAllowedOrigins(Arrays.asList(
        "http://localhost:4200",
        "https://your-frontend-name.vercel.app"  // Add this
    ));
    config.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE", "OPTIONS"));
    config.setAllowedHeaders(Arrays.asList("*"));
    config.setAllowCredentials(true);

    UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
    source.registerCorsConfiguration("/**", config);
    return source;
}
```

### 6.3. Redeploy

```bash
git add .
git commit -m "feat: connect to Render backend"
git push origin main
```

Render sẽ tự động redeploy trong 5-10 phút.

## 🐛 Troubleshooting

### ❌ Build failed: "Cannot download dependencies"

**Lỗi**: Maven không download được dependencies

**Giải pháp**:

```dockerfile
# Trong Dockerfile, thêm timeout
RUN mvn dependency:go-offline -B -Dmaven.wagon.http.connectionTimeout=60000
```

### ❌ Out of Memory (OOM)

**Lỗi**: `java.lang.OutOfMemoryError: Java heap space`

**Giải pháp**: Tối ưu JAVA_OPTS

```bash
JAVA_OPTS=-Xms256m -Xmx450m -XX:MaxRAMPercentage=70.0 -XX:+UseG1GC
```

### ❌ Database connection timeout

**Lỗi**: `Communications link failure`

**Giải pháp**:

1. Kiểm tra DB_URL có đúng không
2. Thêm timeout vào connection string:
   ```
   jdbc:mysql://host:port/db?connectTimeout=30000&socketTimeout=30000
   ```

### ❌ "Web service is sleeping" (Free tier)

**Lỗi**: Service ngủ sau 15 phút không hoạt động

**Giải pháp**:

1. **Upgrade lên Starter plan** ($7/month) - không ngủ
2. **Hoặc dùng cron job** để ping service mỗi 14 phút:
   ```bash
   # Dùng cron-job.org hoặc GitHub Actions
   */14 * * * * curl https://your-app.onrender.com/actuator/health
   ```

### ❌ CORS error từ frontend

**Giải pháp**: Xem Bước 6.2 - Update CORS configuration
