## 🐳 Bước 2: Deploy Backend

### Option 1: Deploy from GitHub (Recommended)

#### 2.1. Push code lên GitHub

```bash
cd library-management-system
git add .
git commit -m "feat: ready for Railway deployment"
git push origin main
```

#### 2.2. Create New Project

1. Vào Railway Dashboard: https://railway.app/dashboard
2. Click **"New Project"**
3. Chọn **"Deploy from GitHub repo"**
4. Select repository: `library-management-system`
5. Railway tự động detect **Dockerfile**!

#### 2.3. Configure Service

Railway tự động:

- ✅ Detect `lms-backend/Dockerfile`
- ✅ Build Docker image
- ✅ Deploy container
- ✅ Generate public URL: `https://your-app.railway.app`

**Customize settings** (nếu cần):

```
Root Directory: lms-backend
Dockerfile Path: Dockerfile
Build Context: lms-backend
Port: 8080
```

## 🗄️ Bước 3: Add MySQL Database

### 3.1. Add MySQL Plugin

1. Trong Project, click **"+ New"** → **"Database"** → **"Add MySQL"**
2. Railway tự động:
   - Provision MySQL 8 instance
   - Generate connection info
   - Set environment variables: `MYSQL_URL`, `MYSQL_HOST`, `MYSQL_PORT`, etc.

### 3.2. Import dữ liệu

```bash
# Lấy connection info từ Railway Dashboard
# Variables tab → Copy MYSQL_URL

# Connect và import
mysql -h <MYSQLHOST> -u <MYSQLUSER> -p<MYSQLPASSWORD> -P <MYSQLPORT> <MYSQLDATABASE> < lms_db_backup.sql
```

**Hoặc dùng Railway CLI:**

```bash
# Install Railway CLI
npm i -g @railway/cli

# Login
railway login

# Link to project
railway link

# Connect to MySQL
railway connect mysql

# Import SQL
SOURCE lms_db_backup.sql;
```

## 🔐 Bước 5: Environment Variables

### 5.1. Configure Variables

Vào **Backend Service** → **Variables** tab:

```bash
# Database (auto-set by Railway)
DATABASE_URL=mysql://root:password@mysql.railway.internal:3306/railway
MYSQL_URL=mysql://root:password@mysql.railway.internal:3306/railway
MYSQLHOST=mysql.railway.internal
MYSQLPORT=3306
MYSQLDATABASE=railway
MYSQLUSER=root
MYSQLPASSWORD=<auto-generated>

# Convert to Spring Boot format
DB_URL=jdbc:${MYSQL_URL}
DB_USERNAME=${MYSQLUSER}
DB_PASSWORD=${MYSQLPASSWORD}

# JWT Secret (generate new!)
APP_JWT_SECRET=your-random-32-bytes-secret-key-here

# Redis (if added)
REDIS_URL=redis://default:password@redis.railway.internal:6379
REDIS_HOST=redis.railway.internal
REDIS_PORT=6379

# Spring Profile
SPRING_PROFILES_ACTIVE=prod

# JVM Optimization
JAVA_OPTS=-Xms256m -Xmx512m -XX:+UseG1GC

# Optional: Gemini AI
GEMINI_API_KEY=your-gemini-key

# Optional: Email
MAIL_HOST=smtp.gmail.com
MAIL_PORT=587
MAIL_USERNAME=your-email@gmail.com
MAIL_PASSWORD=your-app-password
```

### 5.2. Reference Variables

Railway supports variable references:

```bash
# Use Railway's auto-generated MYSQL_URL
DB_URL=jdbc:${MYSQL_URL}

# Or construct manually
DB_URL=jdbc:mysql://${MYSQLHOST}:${MYSQLPORT}/${MYSQLDATABASE}
```

## ✅ Bước 6: Deploy & Test

### 6.1. Trigger Deploy

Railway tự động deploy khi:

1. Push code to GitHub → Auto deploy
2. Change variables → Click **"Deploy"**

**Watch Deployment:**

1. Vào **Settings** tab
2. Click **"Generate Domain"**
3. Railway tạo URL: `https://your-app.up.railway.app`

1. Add your domain: `api.yourdomain.com`
2. Update DNS CNAME record
3. Railway tự động provision SSL

```bash

# Login endpoint
curl -X POST https://your-app.up.railway.app/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@example.com","password":"admin123"}'
```

## 🐛 Troubleshooting

### ❌ Build failed: Docker build error

**Lỗi**: `ERROR: failed to solve`

**Giải pháp**:

1. Kiểm tra Dockerfile syntax
2. Test local: `docker build -t test-image ./lms-backend`
3. Check Railway build logs

### ❌ Database connection timeout

**Lỗi**: `Communications link failure`

**Giải pháp**:

```bash
# Use Railway's internal network
DB_URL=jdbc:mysql://mysql.railway.internal:3306/railway

# NOT public URL
# DB_URL=jdbc:mysql://mysql-abc.railway.app:3306/railway (❌ WRONG)
```

### ❌ Out of Memory

**Lỗi**: `java.lang.OutOfMemoryError`

**Giải pháp**:

```bash
# Optimize JVM for 512MB RAM
JAVA_OPTS=-Xms256m -Xmx450m -XX:MaxRAMPercentage=70.0 -XX:+UseG1GC

# Or increase Railway plan
Settings → Resources → Increase Memory
```

### ❌ Port binding error

**Lỗi**: `Port 8080 already in use`

**Giải pháp**: Railway auto-assigns `$PORT` variable

```java
// application.properties
server.port=${PORT:8080}
```

### ❌ Exceeded free credits

**Giải pháp**:

1. Check **Usage** tab → See what's consuming credits
2. Options:
   - Add payment method (charges after $5)
   - Optimize resource usage
   - Pause services when not needed

## 💰 Pricing & Usage

### Free Tier

- **$5 FREE credits/month** (with email verification)
- Usage-based pricing:
  ```
  CPU: $0.000463/minute (~$0.028/hour)
  RAM: $0.000231/GB/minute (~$0.014/GB/hour)
  ```

### Example Cost (24/7 hobby app)

```
Backend (512MB RAM, 0.5 vCPU):
- CPU: $0.028/hour × 720 hours = $20.16/month
- RAM: $0.014/GB/hour × 0.5GB × 720h = $5.04/month
Total: ~$25/month

MySQL: ~$5/month
Redis: ~$3/month

Total: ~$33/month (but $5 FREE, so $28/month)
```

💡 **Tips to reduce costs:**

- Use smaller RAM (256MB if possible)
- Scale to zero when not used (Pro feature)
- Use external free database (PlanetScale, Neon)

- Includes $5 usage credits
